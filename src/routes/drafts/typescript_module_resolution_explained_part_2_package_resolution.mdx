In the last post we got into the nuance of module resolution for relative (e.g. `"./foo.js"`) imports.

Now let's turn to package resolution - the behaviour of Node as it relates to resolving imports like: 

```
import foo from "mypackage";
```

or 

```
const foo = require("mypackage");
```

This section is, in my opinion, a lot simpler. There weren't any surprises here for me. 

The main reason we want to talk about this is that it's important to know about the `exports` property of the package.json.
