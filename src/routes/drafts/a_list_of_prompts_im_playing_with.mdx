**"Do X task. Do not read any files. Based on your current understanding, how would you achieve this task?"**

I've used it? ✅
I recommend it? 💯

This works really well. 

Firstly, the AI immediately replies, so you know if it's on track or not. 

Secondly, in my case, it turns out the AI already does have everything it needs. I have a file structure listed in my base CLAUDE.md file, I have terminology and examples. 


Prompt: 
```
Implement a 'reflect' compute node.                                                                                 
                                                                                                                      
  This node takes as inputs:                                                                                          
  inputPoints: Array<ColorPoint>                                                                                      
  reflectionPoints: Array<ColorPoint>                                                                                 
                                                                                                                      
  ouputs:                                                                                                             
  points: Array<ColorPoints>                                                                                          
                                                                                                                      
  Each point in the inputPoints will be reflected about each point in the reflection point, where the angle of        
  incidence equals the angle of reflection.                                                                           
                                                                                                                      
  Do not read any files.                                                                                              
                                                                                                                      
  Based on your understanding how would you achieve this task?                                                        
  

```

Result:
```
⏺ Here's how I'd approach this:

  The maths

  Reflecting point P about point R (point symmetry / central reflection): P' = 2R - P. This satisfies angle of
  incidence = angle of reflection. For N input points and M reflection points, the output has N×M reflected points.

  For the colour of each reflected point I'd carry over the colour from the input point (the reflected point is the
  same "thing", just repositioned).

  The steps

  1. Schema agent — define the reflect node in schema.json. Inputs inputPoints and reflectionPoints are
  colorPointArrayValue refs; output points is also colorPointArrayValue. I'll need to check whether
  colorPointArrayValue already exists, and if not, define it in value-kinds.schema.json first, then run bun generate.
  2. Compute node agent — implement the node in src/nodes/compute/nodes/. The evaluate function iterates over all
  pairs (inputPoint, reflectionPoint), computes {x: 2*r.x - p.x, y: 2*r.y - p.y}, and collects results into an output
  array.
  3. Algorithm agent — declare a minimal reference algorithm in src/algorithms/reference/node_specific/ that exercises
   the new node.

```

The formatting of the result isn't quite as nice as I would like, it hasn't told me exactly which files are going to be touched - but this is also a bit on me - in it's CLAUDE.md it's also been told to delegate to sub agents. 

None the less:

```
  Likely, if colorPointArrayValue doesn't already exist:
  - src/schema/schema/value-kinds.schema.json — define the array value type
  - src/schema/schema/refable-value-kinds.schema.json — derived file, may be auto-generated by bun generate
  - Generated TypeScript typings (wherever bun generate writes them)
  
  Possibly, depending on how nodes are registered:
  - An index or registry file under src/nodes/compute/nodes/ — many architectures require you to register new node

  The uncertain ones are the registration file and whether colorPointArrayValue already exists. The agents will need
  to check those.
  ```

**"Sign the guest book."**

Add these to your CLAUDE.md files about the place. Then, if an agent is reading files in that area, it'll sign the guestbook, so you get a picture of what agent is doing. 

I've used it? ✅
I recommend it? 🫤 Not really. It slowed things down a lot, the agent constantly signing guest books. And really, you can find this information out by just watching the output log.
